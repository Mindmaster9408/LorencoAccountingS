# VAT Engine Future Roadmap

## 1. VAT Engine Vision

Authoritative server-side VAT201 engine built from posted journals.

## 2. VAT Basis

Current = invoice basis.
Future = optional cash basis per company if legally applicable.

## 3. Immutable VAT Snapshots

When period is locked/submitted:

- freeze VAT201 numbers
- store source journal IDs
- store generated by/at
- later changes become out-of-period adjustments

## 4. Out-of-Period Adjustment Queue

Future:

- detect locked-period VAT changes
- hold in adjustment queue
- accountant approves period to include adjustment

## 5. SARS Submission Gate

Future submit requires:

- VAT control accounts reconcile
- source journals traceable
- no unresolved warnings
- authorized approval

## 6. Manual Overrides

Allowed only with:

- reason
- approver
- audit log
- visible override layer
- never overwrites source truth

## 7. VAT201 Drilldown

Every VAT201 field must drill down to:

- journal
- invoice/bill/bank allocation
- VAT setting
- account
- period assignment

## 8. Multi-Tenant VAT Safety

No cross-company VAT data exposure ever.
