## 1. Summary

Codebox 07 builds the Compliance Calendar and Deadline Engine foundation for Lorenco Practice Management. This makes the app operationally useful for an accounting practice by providing:

- A structured calendar view of client compliance deadlines
- A list view with rich filtering
- A compliance rule library (what obligations the practice tracks)
- Client-based deadline suggestions derived from compliance registration flags
- Audit trail for deadline status changes
- Enhanced deadline CRUD with compliance fields, assignment, and priority

**What was NOT built (future codeboxes):**
- Cron/scheduler for auto-generating deadlines
- SARS or CIPC API integrations
- Individual or Provisional Tax calculations
- Sean AI
- Cross-app integrations

## 2. Compliance Architecture

```
practice_compliance_rules   — What obligations exist (reference table)
        ↓ (informs)
practice_clients            — Which clients have which flags
        ↓ (GET /compliance/suggestions/client/:id)
Suggestion Engine           — Returns suggested deadlines per client
        ↓ (user explicitly confirms each one)
practice_deadlines          — The actual deadline record (extended schema)
        ↓ (status changes logged)
practice_deadline_events    — Append-only audit trail per deadline
```

---

## 3. Difference Between Rules / Suggestions / Deadlines

| Concept | Table | Purpose |
|---|---|---|
| Compliance Rule | `practice_compliance_rules` | Documents that a practice manages a given obligation type (e.g. "monthly VAT201"). Foundation only — no automation yet. |
| Suggestion | None (derived) | A recommendation generated by `GET /compliance/suggestions/client/:id` based on the client's registration flags. Never auto-created. |
| Deadline | `practice_deadlines` | A concrete, dated compliance task for a specific client. Created manually or from a suggestion. Has status, assignee, audit trail. |

## 8. Suggestion Engine Logic

The suggestion engine (`GET /compliance/suggestions/client/:clientId`) inspects client fields and returns suggestions:

| Client Flag | Suggestion Generated |
|---|---|
| `vat_registered = true` | VAT201 Monthly Return |
| `paye_registered = true` | EMP201 Monthly + EMP501 Annual Reconciliation |
| `provisional_taxpayer = true` | IRP6 P1 + IRP6 P2 |
| `client_type = 'individual'` | ITR12 Individual Tax Return |
| `client_type IN (company, cc, trust, partnership)` | ITR14 Company Return + Annual Financial Statements |
| `cipc_registered = true` | CIPC Annual Return + Beneficial Ownership Register |
| `uif_registered OR paye_registered` | Monthly Payroll Processing |

**Critical rule:** Suggestions are returned as data only. Nothing is auto-created. The user clicks "+ Create Deadline" per suggestion to explicitly create each one.

## 11. Validation Rules

### Deadline validation (backend)
- `title` and `due_date` required
- `compliance_area` must be in allowed list if provided
- `deadline_type` must be in allowed list if provided
- `priority` must be low/normal/high/urgent if provided
- `period_start` ≤ `period_end` if both provided
- `client_id` verified as belonging to `req.companyId`
- `responsible_team_member_id` verified as belonging to `req.companyId`
- Status transitions: `submitted_at` auto-set on submitted, `completed_at`/`completed_by` auto-set on completed
- Cannot edit a cancelled deadline (is_active = false)

### Compliance rule validation (backend)
- `rule_name` required
- `compliance_area` required and in allowed list
- `deadline_type` required and in allowed list
- `recurrence_type` in allowed list if provided
- `due_offset_basis` in allowed list if provided

## 12. Multi-Tenant Safety Review

All deadline and compliance routes use `req.companyId` sourced from JWT via `authenticateToken` middleware.

- Every Supabase query is scoped with `.eq('company_id', req.companyId)`
- `company_id` is never accepted from the request body — always sourced from JWT
- Foreign key ownership verified before use: client_id, responsible_team_member_id
- Suggestions endpoint verifies client.company_id = req.companyId before returning data
- No cross-company leakage possible through the route design

## 18. Recommended Codebox 08

**Workflow-to-Deadline Linking**

Workflows and compliance deadlines now exist separately. Codebox 08 should link them so that:
- Workflow templates can optionally define a deadline type and compliance area
- When a workflow run is generated for a client, a corresponding deadline is automatically created and linked via `workflow_run_id`
- The compliance calendar then reflects which deadlines have associated workflow runs (and their task completion status)

This does not require cron automation — the linking happens at the point of manual workflow generation.
